Real-Time Queue Monitoring

Near-live queue visibility for outbound call-back work, designed for continuous operational use by team leaders.

What I Did

  • Built status reconstruction logic from multiple audit events to show each case's state
  • Created the real-time queries into the case management system to gather the data
  • Developed DAX measures for queue volume, active workload and elapsed time
  • Designed and built a wallboard layout optimised for continuous display
  • Implemented bookmark navigation between queue views and drill-through to case-level detail

Context

A high-volume section of a contact centre performing outbound calls to follow up on earlier inbound conversations. Cases waited in a backlog and were called when an agent was available. Calls had several types which reflected a priority and three or four teams with specific skills performed the call back.

Technologies were T-SQL for the warehouse and native tables and Power BI Direct Query for the semantic model, DAX measures and reporting canvas.

Data Model

Case details were held in the case management system tables, but the queue state had to be inferred from a series of audit events showing movement between queues, entering or exiting the queue or starting or ending a call-back. It could not be read from a single status, and reconstruction logic was written to enable a single status to be observed by the wallboard.

The semantic model was intentionally kept lightweight. A single operational fact table, populated by a near-live SQL extraction process, carried all queue, status, elapsed-time and clinician fields. Small lookup dimensions handled priority labels and queue classifications. Direct Query was used throughout to ensure the wallboard reflected current operational state rather than a cached snapshot. Despite the complexity of the operational logic, the extraction query consistently executed in under one second.

Power BI Design

The Power BI solution was designed as a wallboard rather than a traditional analytical report. The dashboard provided live waiting volumes, active agent workload, longest waiting case by priority, longest active assessment duration and active agent counts by role.

Additional functionality included drill-through into detailed case-level records, bookmark navigation between queue types, and metric definitions and support/version information for governance.

Outcome

The wallboard was deployed for continuous 24/7 operational use across all teams and became the primary operational view of live queue demand across the organisation.
